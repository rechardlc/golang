# TwoSum 语法笔记 (Go vs JS)

这份文档总结了在实现 `twoSum.go` 过程中涉及到的 Go 核心语法，特别为从 JS 转 Go 的开发者准备。

## 1. Map 的操作

### 初始化
- **Go**: `m := make(map[int]int)`
- **JS**: `const m = new Map()` 或 `const m = {}`
> **注意**: 在 Go 中，`map[KeyType]ValueType` 明确了键值的类型。`make` 是必须的，否则 map 是 `nil`，写入会崩溃。

### 写入
- **Go**: `m[v] = i`
- **JS**: `m.set(v, i)` 或 `m[v] = i`
> **注意**: Go 使用普通的 `=` 赋值，因为 `m` 已经声明过了。只有创建新变量时才用 `:=`。

---

## 2. Comma-ok 断言与 `if` 局部作用域

### 语法：`if idx, ok := m[key]; ok { ... }`

这是 Go 的特色组合拳，拆解如下：

1. **`idx, ok := m[key]`**: 
   - `idx`: 拿到的值（如果不存在则是零值）。
   - `ok`: 布尔值，标识 key 是否真的存在 (`true`/`false`)。
2. **中间的 `;`**: 分隔初始化语句和判断条件。
3. **末尾的 `ok`**: 真正的 `if` 判断条件。只有当 `ok` 为 `true` 时才进入代码块。

**JS 对应逻辑**:
```javascript
{
    const idx = m.get(key);
    const ok = m.has(key);
    if (ok) {
        // ...
    }
} // idx 和 ok 在块外不可见
```

---

## 3. 循环与迭代 (`range`)

### 语法：`for i, v := range nums { ... }`

- **Go**: `i` 是索引，`v` 是对应的值。
- **JS**: `nums.forEach((v, i) => ...)` 或 `for (const [i, v] of nums.entries())`

> **注意**: 
> - 如果不需要索引，必须显式忽略：`for _, v := range nums`。
> - 在 Go 中， unused 变量会导致编译错误。

---

## 4. 控制流的极简主义

- **不带小括号**: `if condition { }` 而不是 `if (condition) { }`。
- **强制大括号**: 哪怕只有一行代码，也必须写 `{ }`。
- **强制换行**: `{` 必须跟在 `if` 或 `for` 的同一行。

---

## 5. 返回值与类型字面量

### 语法：`[]int{idx, i}`

- **Go**: 必须带上类型声明 `[]int{...}`。
- **JS**: 直接 `[idx, i]`。

> **nil**: Go 中的 `nil` 类似于 JS 的 `null`，表示切片的空值。

---

## 6. 内置函数 `len`

### 语法：`len(object)`

- **Go**: `len(nums)`
- **JS**: `nums.length` 或 `map.size`

> **注意**: 
> 1. **这不是属性**: 在 Go 中，`len` 是内置函数，不是对象的属性 (不用写成 `nums.len`)。
> 2. **nil 安全**: 如果 `nums` 为 `nil`，`len(nums)` 会返回 `0` 而不会报错。这比 JS 里的 `null.length` 报错要安全得多。
> 3. **字符串坑**: 对于字符串，`len` 返回的是**字节数**。对于中文字符，一个字通常占 3 个字节。
> 4. **性能**: `len` 的时间复杂度是 $O(1)$，极其高效。

---

## 7. 关于分号 `;`

Go 的编译器会自动插入分号，因此在源代码中大部分地方**不需要手动书写**。

### 必须写的情况
- **单行多语句**: `a := 1; b := 2` (虽然不推荐)。
- **控制流头部**: 用来分隔初始化语句和判断/迭代逻辑 (这是最常见的用法)。
  - `if idx, ok := m[key]; ok { ... }`
  - `for i := 0; i < 10; i++ { ... }`

### 不要写的情况
- **行收尾**: Go 会根据换行符自动识别语句结束，手动加分号会被 `gofmt` 强制删除。

### 强制换行规则
由于 Go 的分号自动补全机制，`{` **必须放在行末**。不能另起一行，否则编译器会在上一行末尾插入分号，导致代码结构被切断而报错。
