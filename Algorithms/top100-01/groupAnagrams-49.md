# GroupAnagrams 语法笔记 (Go vs TS)

这份文档总结了在实现 `groupAnagrams-49.go` 过程中涉及到的 Go 核心语法和工程实践，特别针对从 TS/JS 转 Go 的开发者。

## 1. 数组作为 Map 的 Key

这是 Go 与 TS/JS 最显著的区别之一。

- **Go**: `map[[26]int][]string`
- **TS**: `Map<string, string[]>` (通常需要 `array.join(',')` 序列化)

> [!IMPORTANT]
> **值类型对比**：
> 在 Go 中，固定长度的数组 `[N]T` 是**值类型**。只要两个数组的元素完全一样，它们就是“相等的”。因此，数组可以直接作为 Map 的 Key。
> 在 TS 中，数组是**引用类型**，即使内容相同，引用地址不同也会被视为不同的 Key。

---

## 2. 字符与 ASCII 运算

### 语法：`cnt[s[i]-'a']++`

- **原理**：利用字符在内存中的数值（byte/uint8）进行偏移计算。
- **拆解**：
    - `s[i]`：获取字符串第 $i$ 个字节（ASCII 码）。
    - `'a'`：小写字母 a 的 ASCII 码 (97)。
    - `s[i] - 'a'`：将 'a'-'z' 映射到 `0-25` 的索引区间。

**TS 对应逻辑**:
```typescript
cnt[s.charCodeAt(i) - 97]++;
```
> Go 的处理更加底层且直观，直接支持字符常量参与减法。

---

## 3. 切片 (Slice) 的追加与零值特性

### 语法：`mp[cnt] = append(mp[cnt], s)`

这是 Go 中极具代表性的地道写法（Idiomatic Go）。

1. **`append` 函数**：非原地操作。它接收一个切片，返回一个**可能已经扩容**的新切片。必须写回原变量：`s = append(s, val)`。
2. **利用零值**：
    - 当 `mp[cnt]` 不存在时，它会返回 `[]string` 的零值：`nil`。
    - `append(nil, s)` 是合法的，会初始化并返回 `["s"]`。
    - 这省去了 TS 中繁琐的 `if (!map.has(key)) map.set(key, [])` 判断。

---

## 4. `make` 的深度使用

### 语法：`res := make([][]string, 0, len(mp))`

- **参数 1**: 类型 `[][]string`。
- **参数 2 (Length)**: `0`。因为我们要通过 `append` 往里塞，初始长度必须为 0。
- **参数 3 (Capacity)**: `len(mp)`。预先分配底层空间。

> [!TIP]
> **性能优化**：
> 在已知最终元素个数的情况下，始终提供 **Capacity**。这能避免 `append` 过程中频繁的内存重新申请和数据拷贝。

---

## 5. 排序操作 (`sort` 包)

### 语法：`sort.Slice(b, func(i, j int) bool { ... })`

- **包引入**：必须显式 `import "sort"`。
- **原地排序**：直接修改传入的切片 `b`。
- **索引比较**：回调函数的参数是**索引** `i, j`，而不是元素本身。

**TS 对比**:
```typescript
b.sort((valA, valB) => valA - valB); // 参数是值
```

---

## 6. 算法策略对比

| 策略 | 性能 | 特点 |
| :--- | :--- | :--- |
| **计数法 (Counting)** | **极高** | $O(N \times K)$，利用数组 Key 省去序列化开销，内存友好。 |
| **排序法 (Sorting)** | **一般** | $O(N \times K \log K)$，逻辑直观，但涉及 `[]byte` 与 `string` 的强制转换。 |

> 在 Go 中，由于字符串是不可变的（immutable），排序时必须经历 `string -> []byte -> sort -> string` 的过程，这会产生额外的内存分配需求。

---

## 7. 循环策略：`range` vs `for i`

遍历字符串或切片时的常见选择。

### **方式 A：`range` 迭代**
```go
for i, v := range s { ... } // 同时拿到索引和值
for i := range s { ... }    // 仅拿到索引
```
- **行为**：如果是字符串，`range` 会尝试进行 **UTF-8 解码**。
- **返回值**：`i` 是字节偏移量，关键是 `v` 是 `rune` (int32)，即完整的 Unicode 字符。
- **索引跳跃**：处理多字节字符（如中文）时，`i` 会跳过中间字节（如 `0, 3, 6...`）。

### **方式 B：传统 `for i` 循环**
```go
for i := 0; i < len(s); i++ { ... }
```
- **行为**：**纯字节访问**。完全不关心字符编码。
- **复杂度**：极快，直接读取内存。
- **局限性**：处理非 ASCII 字符（如中文）会产生乱码（只取到 1/3 个字符）。

**TS 视角建议**:
- 遍历**数组/切片**：优先用 `range` (类似 `for...of`)。
- 遍历**字符串**（且涉及 ASCII 运算）：优先用 `for i`，它最接近 C/JS 处理单字节逻辑的直觉，且性能最高。
